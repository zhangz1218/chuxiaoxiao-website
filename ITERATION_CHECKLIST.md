# 网站迭代 Checklist

每次迭代网站时，按此流程执行：

## 迭代前

- [ ] 思考：这次迭代要解决什么问题？
- [ ] 备份：git tag backup-YYYY-MM-DD-HHMM
- [ ] 保留最近 3-5 个备份，确认无问题（2-3天）后删除旧备份

## 迭代中

- [ ] 本地开发测试：npm run dev
- [ ] 确认功能正常，无报错

## 迭代后（必须按顺序执行）

1. [ ] 推送代码：git add . && git commit -m "message" && git push
2. [ ] 等待 Vercel 自动部署（1-2分钟）
3. [ ] **发飞书通知拯哥**：说明改了什么、为什么改
4. [ ] 更新 MEMORY.md（如有重要经验）

## 迭代频率

- 当前阶段：每 2-3 小时思考一次优化
- 未来调整：网站稳定后可降低频率

## 必须功能

- 每天自动同步日记到网站

---

**重要**：第3步"发飞书通知"不可跳过！拯哥需要知道网站在持续迭代。
